# تحسينات نظام الاختبارات

## التغييرات المنفذة

تم تحسين نظام الاختبارات لإضافة محاولات جديدة وتقسيم المراحل بشكل أفضل.

## 1. إضافة محاولات جديدة

### المحاولات المتاحة لجميع المستويات:
- **المحاولة الأولى** (1)
- **المحاولة الثانية** (2)
- **المحاولة الثالثة** (3)
- **المحاولة الرابعة** (4)
- **المرحلة** (100) - جديد ✨

### محاولة خاصة بالحفاظ فقط:
- **التثبيت** (200) - جديد ✨

## 2. تقسيم المراحل (كل 5)

- المرحلة السادسة (26-30)

## 3. تحديث عرض المحاولات

تم تحديث دالة `getAttemptText` في `ExamSection.tsx` لعرض المحاولات الجديدة:

```typescript
const getAttemptText = (attemptNumber: number) => {
  switch (attemptNumber) {
    case 1: return "المحاولة الأولى";
    case 2: return "المحاولة الثانية";
    case 3: return "المحاولة الثالثة";
    case 4: return "المحاولة الرابعة";
    case 100: return "المرحلة";
    case 200: return "التثبيت";
    default: return `المحاولة ${attemptNumber}`;
  }
};
```

## 4. الفوائد

### محاولة "المرحلة" (100):
- تُستخدم لاختبارات المراحل الشاملة
- متاحة لجميع المستويات (تمهيدي، تلاوة، حافظ)
- تساعد في تتبع تقدم الطالب عبر المراحل الكبرى

### محاولة "التثبيت" (200):
- خاصة بطلاب الحفظ فقط
- تُستخدم لاختبارات التثبيت والمراجعة الشاملة
- تساعد في قياس ثبات الحفظ لدى الطالب

### تقسيم المراحل كل 5:
- تنظيم أفضل للمراحل
- تتبع دقيق لتقدم الطالب
- سهولة في تحديد المستوى الحالي للطالب
